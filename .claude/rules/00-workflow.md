# Правила работы

## Язык
- Общение и комментарии в коммитах — на русском.

## Docker и инфраструктура
- **Не запускать** `docker`/`docker compose` напрямую. Только `./manage.sh`:
  - `<dev|prod> up/down/restart/build` — управление контейнерами
  - `<dev|prod> logs` — логи (tail 100)
  - `<dev|prod> exec <front|back|proxy> [cmd]` — выполнить команду в контейнере
  - `<dev|prod> npm-install <front|back> [args]` — npm install через временный Node-контейнер
  - `<dev|prod> init [back|front]` — инициализация (папки, секреты); **только ручной запуск владельцем**
  - `ps` — список запущенных контейнеров
- **Режим `prod` не использовать локально.** Он предназначен для ручных манипуляций на продакшн-сервере. Здесь работать только с `dev`.
- **Не изменять** `compose*.yaml`, Dockerfile'ы, конфиги Caddy без явного указания. Если задача требует изменений в инфра — дать инструкцию владельцу; он вносит сам.

## Секреты
- Реальные секреты — только в `.env.local` (gitignored), не в `.env`.
- `.env` содержит только dev-дефолты, безопасные для коммита.

## npm
- `npm install`, `npm run ...` можно запускать на хосте в `front/` или `back/` — директории смонтированы в контейнеры.
- Учитывать, что код исполняется внутри контейнера с Node 24; выбирать совместимые зависимости.

## Проблемы и обходные пути
- При нестандартных трудностях (команда не работает, инструмент не отвечает, непонятное поведение) — **остановиться и спросить**, а не искать обходной путь самостоятельно.
- Если всё же пришлось решить проблему обходным путём — обязательно сообщить: что пошло не так, что было сделано иначе и почему.
- Если обходной путь затронул архитектуру или инфраструктуру — сообщить особо явно.
